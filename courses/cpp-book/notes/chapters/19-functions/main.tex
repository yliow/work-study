Note that the body of the function is just like the code you write in \texttt{main()}. (In fact \texttt{main()} \textbf{\textit{is}} a function!) You can have as many statements as you like as long as they are valid statements.

Let's get C++ to show us the flow of execution into and out of the function. Modify the previous program and run it:

\begin{consolethree}[escapeinside=||]
#include <iostream>

int square(int x)
{
    std::cout << "in square() ... "
              << "x = " << x << std::endl;
    int y = x * x;
    std::cout << "exiting square() ... "
              << "returning " << y << std::endl;
    return y;

int main()
{
    std::cout << "calling square() with 3 ...";
    int a = square(3);
    std::cout << "... back from square() ... ";
    std::cout << "received " << a << std::endl;

    int i = 5;
    std::cout << "calling square() with 5 ...";
    std::cout << square(i) << std::endl;

\newpage\EMPHASIZE{Pass-by-Value}

It's extremely important to remember that \texttt{main()} calls \texttt{square()} by passing in a value. This is called \textbf{pass-by-value}. The parameter (i.e. \texttt{x}) receives the value.

Here's a \textbf{very common gotcha}: If \texttt{main()} calls \texttt{square()} with a variable say \texttt{i}, and \texttt{x} (in \texttt{square()}) receives the \textbf{value} of \texttt{i}, then modifying \texttt{x} (in \texttt{square()}) does NOT affect the value of \texttt{i}.

Here's an example where the value of \texttt{i} is passed in to our \texttt{square()} function with \texttt{x} receiving the value. We change the value of \texttt{x} and back in \texttt{main()}, then value of \texttt{i} is not affected.

\begin{consolethree}[escapeinside=||]
#include <iostream>

int square(int x)
{
    int y = x * x;
    x = 0;
    return x * x;
}

int main()
{
    int i = 5;
    std::cout << square(i) << std::endl;
    std::cout << i << std::endl;

    return 0;
}
\end{consolethree}

Many students think that somehow the \textbf{variable} \texttt{i} (in \texttt{main()}) is passed in and that \texttt{x} (in \texttt{square()}) is somehow tied to the i (in \texttt{main()}).

That's \EMPHASIZE{NOT TRUE}. Only the value of \texttt{i} (in \texttt{main()}) is passed onto the \texttt{x} (in \texttt{square()}). \EMPHASIZE{The two variables have their own memory spaces. Changing one does not change the other.}

It's also important to remember that the memory of \texttt{square()} is created when you call it. When you return to \texttt{main()} (i.e. when you exit the block for \texttt{square()}), the memory for that particular call of \texttt{square()} is destroyed.

The following example shows you that when you call \texttt{square()}, the memory created for that call is different from the memory of \texttt{main()}. We'll create a variable \texttt{i} in \texttt{square()} and set it to zero. The \texttt{i} (in \texttt{main()}) is not changed.

\begin{consolethree}[escapeinside=||]
#include <iostream>

int square(int x)
{
    int y = x * x;
    int i = 0;
    return x * x;
}

int main()
{
    int i = 5;
    std::cout << square(i) << std::endl;
    std::cout << i << std::endl;

    return 0;
}
\end{consolethree}

There are several important things to observe in this simulation:

\begin{itemize}
\item The memory for the call to \texttt{square()} is different from the memory of \texttt{main()}
\item Values are passed between \texttt{main()} and \texttt{square()} and not the variables.
\end{itemize}

By the way there's something special about the stack. A \EMPHASIZE{stack} is a data structure (i.e. container of data with operations) that allows you to put things into it (this operation is called \EMPHASIZE{push}) and also to get something out of it (the operation is called \EMPHASIZE{pop}). You cannot choose what you want to get out of the stack: It will always be the last thing that you put into it. That's why the stack is also called a \EMPHASIZE{last-in-first-out} \EMPHASIZE{(LIFO)} data structure.

A stack of plates at a buffet is (almost) a stack. The plate you take (if you only take from the top) is the last plate placed onto the stack.

The above model illustrates many concepts in function calling. But it's only a model. For the real picture you have to take CISS360 (Assembly Language and Computer Systems) and to understand how Physics (or electronics) can be used to physically implement a computational device you have to take CISS430 (Computer Architecture).

\newpage\EMPHASIZE{Pass-by-reference}

Recall that the parameter \texttt{x} in function \texttt{f()} is pass-by-value:

\begin{consolethree}[escapeinside=||]
void f(int x) // x is a variable and has
              // it's own value. It cannot
              // change y in main().
{
    x = 42;
}

int main()
{
    int y = -1;
    f(y); // value of y passed to x
    // y is not 42. y is still -1.
    std::cout << y << std::endl;
    return 0;
}
\end{consolethree}

Note that the \texttt{y} in \texttt{main()} is \EMPHASIZE{NOT} changed when you return back to \texttt{main()}.

Now try this:

\begin{consolethree}[escapeinside=||]
void f(int & x) // x is not a variable -- x
                // is a reference or a
                // reference variable. x
                // does not have a value --
                // x uses the value of y.
                // x can change the y in
                // main().
{
    x = 42;
}

int main()
{
    int y = -1;
    f(y); // y is passed to x in f().
    // y is not -1. y is now 42.
    std::cout << y << std::endl;
    return 0;
}
\end{consolethree}

In the above example, the variables \texttt{x} in \texttt{f()} is called a \EMPHASIZE{reference variable}, in particular \texttt{x} is an integer reference variable. \texttt{x} is different from the regular plain-jane variables: \texttt{x} \EMPHASIZE{does not have its own memory for keeping a value}. Instead, \texttt{x} \textbf{refers} to other variables. In the above example, \texttt{x} refers to the value of \texttt{y} in \texttt{main().}

A reference variable is an \EMPHASIZE{alias} of another variable -- in the above example the \texttt{x} in \texttt{f()} is just another name for the \texttt{y} in \texttt{main()}.

When \texttt{main()} calls \texttt{f()}, the integer reference variable \texttt{x} in \texttt{f()} will \textbf{refer} to the memory of \texttt{y}; it \EMPHASIZE{does not receive a value} from \texttt{y}. As a matter of fact, referring to the diagram, you see that \texttt{x} does not even have its own memory!

We call this type of parameter passing \EMPHASIZE{pass-by-reference}. We say that the \texttt{x} in \texttt{f()} is a pass-by-reference parameter.

So

\begin{itemize}
\item If \texttt{y} in \texttt{main()} is pass to a variable \texttt{x} of \texttt{f()}, changing \texttt{x} in \texttt{f()} does not change the \texttt{y} in \texttt{main()}.
\item If \texttt{y} in \texttt{main()} is pass to a reference \texttt{x} of \texttt{f()}, changing \texttt{x} in \texttt{f()} changes the \texttt{y} in \texttt{main()}.
\end{itemize}

Also

\begin{itemize}
\item If you use an integer variable as a function parameter, the communication into that function uses pass-by-value.
\item If you use a reference as a function parameter, the communication into that function uses pass-by-reference.
\end{itemize}

This will save you some time on re-organizing your code. That's it.

\newpage\EMPHASIZE{Summary}

A function looks like this:

\texttt{ [return type] [func name]}( \texttt{[param type] [var name]} )

\{

\texttt{[body of function]}

\}

There can be more than one parameter type and name. A parameter is just a variable that receives a value passed into the function. The return type can be any valid type including \texttt{void} if the function does not return a value.

The function body can be any valid C++ statement. To return a value the
function executes

\texttt{return [expression];}

in its body. If the function does not return a value, executing
\begin{center}
\texttt{return;}
\end{center}
will return to the point of function call.

If a function has return type void, return need not be present. In that case when the execution reached the end of the function block, a return is executed.

Functions cannot be nested.

The largest scope in a program file is the global scope. You can declare variables and constants in the global scope. These are called global variables and global constants respectively. However you should avoid (if possible) the declaration of global variables.

On returning from a function, since the program leaves the scope of the function, all variables created within the function's scope are destroyed.

A parameter is pass-by-value if the parameter has its own value. A parameter is pass-by-reference if the parameter does not have its own value but references the value of a variable that is passed to that parameter. Changing the value of the parameter that is pass-by-reference will change the value of the variable passed to the parameter. By default, parameters of basic types (int, double, char, bool) are pass-by-value. The following parameter \texttt{x} is pass-by-value:
\begin{center}
\texttt{void f(..., int x, ...) { ... }}
\end{center}
while the following parameter \texttt{y} is pass-by-reference:
\begin{center}
\texttt{void g(..., int \& x, ...) { ... }}
\end{center}
A function prototype is just the header of a function as a statement. A function prototype is usually placed near the top of source files in order to indicate to the compiler the existence that function. If a function prototype is seen by the compiler, then calling that function will be compiled correctly by the compiler even if the function definition is below the point where the function call is made.
