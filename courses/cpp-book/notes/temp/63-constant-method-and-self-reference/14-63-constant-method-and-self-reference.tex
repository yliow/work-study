\sectionthree{Forcing a return reference to be constant}

Recall this example from above:

\begin{consolethree}[escapeinside=||]
class Point2d
{
public:
    int & get_x()
    {
        return x_;
    }
    void print()
    {
        std::cout << x_ << ',' << y_ << '\n';
    }
private:
    int x_, y_;
};

int main()
{
    Point2d p;
    p.print();
    int & a = p.get_x();
    a = 0;
    p.print();
    return 0;
}
\end{consolethree}

The \texttt{p.get\_x()} method returns a reference to \texttt{p.x\_}. This
reference is then given to \texttt{a}:

\begin{consolethree}[escapeinside=||]
int & a = p.get_x();
\end{consolethree}

This means that a is a reference to p.x\_. Therefore changing a will
mean changing p.x\_.

But remember what I said earlier about information hiding. Clients
usually should not have direct access to instance variables. The class
(and the objects) should provide services to clients through methods and
hide internal implementation from outsiders. This makes the software
more maintainable. So to prevent \texttt{main()} from changing
\texttt{p.x\_} directly, you can do this:

\begin{consolethree}[escapeinside=||]
class Point2d
{
public:
    |\EMPHASIZE{const}| int & get_x()
    {
        return x_;
    }
    ...
\end{consolethree}

The reference returned will not be able to change the x\_ of the object.
In fact, now the \texttt{main()} above cannot be compiled. Why?

\begin{consolethree}[escapeinside=||]
class Point2d
{
public:
    const int & get_x()
    {
        return x_;
    }
    ...
};

int main()
{
    Point2d p;
    p.print();
    int & a = p.get_x();
    ...
}
\end{consolethree}

Because the \texttt{a} in \texttt{main()} is a reference that can
potentially attempt to the the value of \texttt{p.x\_}. The issue is not
whether \texttt{a} will actually change or not. The only way for the above
to compile is to make it impossible for a to change, i.e., you would
have to do this:

\begin{consolethree}[escapeinside=||]
class Point2d
{
public:
    const int & get_x()
    {
        return x_;
    }
    ...
};

int main()
{
    Point2d p;
    p.print();
    |\EMPHASIZE{const}| int & a = p.get_x();
    ...
}
\end{consolethree}
