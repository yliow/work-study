\sectionthree{When do you use this?}

\EMPHASIZE{Standard Practice:} Don't use \texttt{this} in a method just to access members or member functions. In other words, in a method

\begin{itemize}
\item \texttt{this->x} should be \texttt{x}
\item \texttt{this->m()} should be \texttt{m()}
\end{itemize}

For instance, this does work:

\begin{consolethree}[escapeinside=||]
...
class Date
{
public:
    ...
    int get_year()
    {
        return |\EMPHASIZE{this->yyyy\_;}|
    }
    void set_year(int y)
    {
        |\EMPHASIZE{this->yyyy\_}| = y;
    }
    ...
    void add_m_y(int m, int d)
    {
        |\EMPHASIZE{this->add\_m(m);}|
        |\EMPHASIZE{this->add\_d(d);}|
    }
private:
    ...
};
\end{consolethree}

but \EMPHASIZE{DON'T DO THAT!!!} Do this:

\begin{consolethree}[escapeinside=||]
...
class Date
{
public:
    ...
    int get_year()
    {
        return |\EMPHASIZE{yyyy\_;}|
    }
    void set_year(int y)
    {
        |\EMPHASIZE{yyyy\_}| = y;
    }
    ...
    void add_m_y(int m, int d)
    {
        |\EMPHASIZE{add\_m(m);}|
        |\EMPHASIZE{add\_d(d);}|
    }
private:
    ...
};
\end{consolethree}

The pointer \texttt{this} is used when it's used on its own either as
\texttt{this} or \texttt{*this}. In particular \texttt{*this} can be used for
returning a copy of or a reference to the object making the method
invocation and \texttt{this} can be used for address comparison
(you'll see this later when we need to compare address
values when we overload the assignment operator \texttt{operator=}.)

(There are exceptions ... later. But the above two uses are good
enough.)
