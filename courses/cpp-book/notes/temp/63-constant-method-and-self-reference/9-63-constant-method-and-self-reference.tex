\sectionthree{Returning reference to member variable}

Of course you can also return a reference to a member (yes I know
I've already talked about this before).

\begin{consolethree}[escapeinside=||]
class Point2d
{
public:
    int & get_x()
    {
        return x_;
    }
    void print()
    {
        std::cout << x_ << ',' << y_ << '\n';
    }
private:
    int x_, y_;
};

int main()
{
    Point2d p;
    p.print();
    int & a = p.get_x();
    a = 0;
    p.print();
    return 0;
}
\end{consolethree}

But remember about information hiding. Clients usually should not have
direct access to instance variables. The class (and the objects) should
provide services to clients through methods and hide internal
implementation from outsiders. This makes the software more
maintainable.

And you definitely cannot do this in a method:

\begin{consolethree}[escapeinside=||]
class Point2d
{
public:
    int & get_x()
    {
        int x = x_;
        return x; // ERROR! Returning reference to
                  // local variable!!! Arghh!!!
    }
private:
    int x_, y_;
};
\end{consolethree}
