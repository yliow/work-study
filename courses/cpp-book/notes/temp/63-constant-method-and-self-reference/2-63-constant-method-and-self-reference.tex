\sectionthree{Thinking of Objects}

You should think of a class as a rubber stamp for creating objects.

You should think of each object as a computational machine.

\begin{python}
from latextool_basic import *
p = Plot()
p += Circle(3, 2, r=2.5, background='red')
p += Circle(3, 2, r=1.15, background='white', label='x\_')
p += Rect(3.25, 1.85, 3.9, 2.15, innersep=0.1, s='42', align='c', linewidth=0.05)

p += Rect(3.25, 3.25, 4, 4, innersep=0.1, s='m1', align='c', linewidth=0.075, linecolor='blue')
p += Rect(0.75, 1.8, 1.65, 2.3, innersep=0.1, s='m2', align='c', linewidth=0.075, linecolor='blue')
p += Rect(3.25, -0.05, 4.4, 0.65, innersep=0.1, s='print', align='c', linewidth=0.075, linecolor='blue')

st = r"""
\begin{consolethree}
Int

- x_  :  int

+ print: void
+ m1: void
+ m2: void
\end{consolethree}
"""
p += Rect(-5, 0, -1, 4, s = st, linewidth=0)
p += Line(points=[(-4.95, 2.5), (-0.95, 2.5)], linewidth=0.01)
p += Line(points=[(-4.95, 3.35), (-0.95, 3.35)], linewidth=0.01)
print(p)
\end{python}
You can (and should) think of the ``stuff'' on the green band (the
\texttt{m1}, \texttt{m2}, \texttt{print}) -- the \texttt{public} things -- as
things that outsiders can access while the stuff on the inside (the
\texttt{x\_}) -- the \texttt{private} things -- are not available to
outsiders. By outsiders, I mean functions not in the class and methods
not in this class (i.e. methods of another class).
