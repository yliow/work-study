\sectionthree{Coding style for blocks}

\textbf{ALIGN BLOCK DELIMITERS TO HEADER:}

BAD (but used by some and is common in Java):
\begin{console}
int main() {
    return 0;
}  
\end{console}
and\\
\begin{console}
if (x < 0) {
   foo = 42;
   bar = 43;
} 
\end{console}

HORRIFYING:\\
\begin{console}
if (x < 0)
   {
   foo = 42;
   bar = 43;
   } 
\end{console}

ABOMINABLE:
\begin{console}
if (x < 0)
   {
        foo = 42;
        bar = 43;
   } 
\end{console}

GOOD:
\begin{console}
int main()
{
    return 0;
}  
\end{console}

and
\begin{console}
if (x < 0)
{
    foo = 42;
    bar = 43;
} 
\end{console}

The same applies to if-else:

BAD
\begin{console}
if (x < 0) {
    foo = 42;
    bar = 43;
}
else {
    foo = 43;
    bar = 42;
}
\end{console}
GOOD
\begin{console}
if (x < 0)
{  
    foo = 42;
    bar = 43;
}
else
{       
    foo = 43;
    bar = 42;
}
\end{console}

\textbf{ALWAYS USE BLOCKS IF NOT ONE-LINERS:}

C/C++ programmers tend to use blocks even when the block has only one
statement except when using the one-line if style. This creates a
unified look and prevents difficult-to-find bugs caused by \lq\lq dangling
else'' (see later).

OK BUT NOT COMMON:
\begin{console}
if (x < 0)
   foo = 42;
else
   foo = 24; 
\end{console}

COMMON:
\begin{console}
if (x < 0)
{
    foo = 42;
}
else
{
    foo = 42;
}
\end{console}
