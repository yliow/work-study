\sectionthree{Application: Special global objects}

In class \texttt{vec2d}, suppose you \textbf{frequently} use a
\texttt{vec2d} object with x value of 0 and y value of 0. Because of its
values, let's call this concept (or object) the origin.

You can create \texttt{origin} as a public static \texttt{vec2d} object in
\texttt{vec2d} class. This allows you do this in \texttt{main()}:

\begin{consolethree}[escapeinside=||]
#include <iostream>
#include "vec2d.h"

int main()
{
    std::cout << vec2d::origin << '\n';

    return 0;
}
\end{consolethree}

which has the same effect as

\begin{consolethree}[escapeinside=||]
#include <iostream>
#include "vec2d.h"

int main()
{
    std::cout << vec2d(0, 0) << '\n';

    return 0;
}
\end{consolethree}

except that the first method is faster since the second method has to
create \texttt{vec2d(0,0)} during runtime and if you need
\texttt{vec2d(0,0)} again and again and again and again it might be slower
(especially if the class in question has a lot more member variables and
consumes a lot more memory.)
