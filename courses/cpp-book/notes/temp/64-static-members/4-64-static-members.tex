\sectionthree{Static methods (i.e. static member functions)}

Just like a regular member variable can be made static (so that it
belongs to the class), a method can also made static.

To make a method static put the keyword \texttt{static} in front of the
prototype of the method in the class declaration.

A static method or static member function is like a method that belongs
to the class and not to individual objects. In general, this is how you
decide if a method should be static or not ...

If a method of a class \texttt{C} does not work with any regular member
variable of \texttt{C}, then the method should be made static.

\input{temp/64-static-members/exercises/1-64-static-members/question.tex}
Here's our class \texttt{C} again:

\begin{consolethree}[escapeinside=||]
// C.h
#ifndef C_H
#define C_H

class C
{
public:
    int x;
    static int y;
    static int z;
};

#endif
\end{consolethree}

\begin{consolethree}[escapeinside=||]
// C.cpp
#include "C.h"

int C::y(42);
int C::z = 99;
\end{consolethree}

Note that in general, you want to hide all member variables, static or
not. So I really should do this:

\begin{consolethree}[escapeinside=||]
// C.h
#ifndef C_H
#define C_H

class C
{
public:
private:
    int x;
    static int y;
    static int z;
};

#endif
\end{consolethree}

\begin{consolethree}[escapeinside=||]
// C.cpp
#include "C.h"

int C::y(42);
int C::z = 99;
\end{consolethree}

What if I might want to print the value of \texttt{C::y}? The following
does not work any more since \texttt{C::y} is now private:

\begin{consolethree}[escapeinside=||]
#include <iostream>
#include "C.h"

int main()
{
    std::cout << C::y << '\n'; // YIKES!!!
    return 0;
}
\end{consolethree}

What should we do?

No problem... you just need a public method to return the value of
\texttt{C::y}:

\begin{consolethree}[escapeinside=||]
// C.h
...
class C
{
public:
    int get_y() const;
private:
    ...
};
...
\end{consolethree}

\begin{consolethree}[escapeinside=||]
// C.cpp
#include "C.h"

...

int C::get_y() const
{
    return y;
}
\end{consolethree}

\begin{consolethree}[escapeinside=||]
#include <iostream>
#include "C.h"

int main()
{
    //std::cout << C::y << '\n';
    C c;
    std::cout << c.get_y() << '\n';
    return 0;
}
\end{consolethree}

Note that in this case, we're accessing \texttt{C::y}
using \texttt{c}, which is a \texttt{C} object. It does work, but note the
\texttt{C::get\_y()} method works only with a static member variable. So
it's better to make \texttt{C::get\_y()} a static method
like this:

\begin{consolethree}[escapeinside=||]
// C.h
...
class C
{
public:
    |{\bf static}| int get_y(); // remove "const"
private:
    ...
};
...
\end{consolethree}

\begin{consolethree}[escapeinside=||]
// C.cpp
#include "C.h"

...

int C::get_y() // remove "const"
{
    return y;
}
\end{consolethree}

\begin{consolethree}[escapeinside=||]
#include <iostream>
#include "C.h"

int main()
{
    std::cout << C::get_y() << '\n';
    return 0;
}
\end{consolethree}

Easy right?

WARNING: Do not include the \texttt{static} keyword in the definition of
the method in the cpp file. (But if the definition of the static method
is in the class within the header file, i.e. it is inlined, then you
have to include the keyword \texttt{static}.)

IMPORTANT WARNING ... Because static methods belong to the class rather
than the object and only work with static member variables, the body of
the method must not refer to the \texttt{this} pointer. There is no
\texttt{this} pointer in the body of a static method. Also, you cannot do
this:

\begin{consolethree}[escapeinside=||]
// C.h
...
class C
{
public:
    static int get_y() const; // Huh?!?
private:
    ...
};
...
\end{consolethree}

Because ``const'' refers to the fact the object making this method call
cannot be changed by this method. But since the method should be called
by the class, there's no object involved (even if you
use a \texttt{C} object to invoke \texttt{get\_y()}).

Although you can also call a static method through an object:

\begin{consolethree}[escapeinside=||]
#include <iostream>
#include "C.h"

int main()
{
    std::cout << C::get_y() << '\n'; // BETTER
    C c;
    std::cout << c.get_y() << '\n'; // WORKS ... BUT
    return 0;
}
\end{consolethree}
