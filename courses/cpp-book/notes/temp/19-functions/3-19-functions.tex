\sectionthree{A mental computational model}

To explain the execution and return of a function call, here's a model of our ``computer''.

We're adding a special container, called the \EMPHASIZE{stack}, that will allow us to do three things:

\begin{itemize}
\item Remember the point of return
\item Put value(s) to pass to the function
\item Put value to pass back from the function
\end{itemize}

Remember that \EMPHASIZE{this is only a model}.

Here's the program we will execute on our model. (We won't talk about \#\verb!include!. That will come soon.)

\begin{consolethree}[escapeinside=||]
#include <iostream>

int square(int x)
{
    return x * x;
}

int main()
{
    int i = 5;
    int j = square(i + 1);
    std::cout << j << std::endl;

    return 0;
}
\end{consolethree}

That's it!

There are several important things to observe in this simulation:

\begin{itemize}
\item The memory for the call to \texttt{square()} is different from the memory of \texttt{main()}
\item Values are passed between \texttt{main()} and \texttt{square()} and not the variables.
\end{itemize}

By the way there's something special about the stack. A \EMPHASIZE{stack} is a data structure (i.e. container of data with operations) that allows you to put things into it (this operation is called \EMPHASIZE{push}) and also to get something out of it (the operation is called \EMPHASIZE{pop}). You cannot choose what you want to get out of the stack: It will always be the last thing that you put into it. That's why the stack is also called a \EMPHASIZE{last-in-first-out} \EMPHASIZE{(LIFO)} data structure.

A stack of plates at a buffet is (almost) a stack. The plate you take (if you only take from the top) is the last plate placed onto the stack.

The above model illustrates many concepts in function calling. But it's only a model. For the real picture you have to take CISS360 (Assembly Language and Computer Systems) and to understand how Physics (or electronics) can be used to physically implement a computational device you have to take CISS430 (Computer Architecture).

By the way, it's actually \EMPHASIZE{OK not to use a return value}. For instance the following program is valid:

\begin{consolethree}[escapeinside=||]
int square(int x)
{
    return x * x;
}

int main()
{
    square(4);
    return 0;
}
\end{consolethree}

As you can see in the above code the value 16 is neither being used in any computation or nor in a print statement. It's just thrown away.

In fact something like this is also valid! (Although quite silly).

\begin{consolethree}[escapeinside=||]
int main()
{
    16;
    return 0;
}
\end{consolethree}

Run it. Again the value 16 is not used in any way.

Note again that when C++ returned from \texttt{square()}, the memory used for \texttt{square()}'s variables is destroyed. This is similar to exiting for instance a for-loop. So in this code segment that uses \texttt{square()} twice:

\begin{consolethree}[escapeinside=||]
...
std::cout << square(5) << std::endl;
...
std::cout << square(i) << std::endl;
\end{consolethree}

the memory for \texttt{square()} to hold data is created and destroyed twice.
