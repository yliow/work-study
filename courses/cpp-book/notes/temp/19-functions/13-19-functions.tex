\sectionthree{Global variables and global constants}

Based on the way C++ search for a variable, can you figure out the output of the following program? Verify your answer.

\begin{consolethree}[escapeinside=||]
#include <iostream>

int y = 1;

void f()
{
    int y = 42;
    std::cout << y << '\n';
}

int main()
{
    std::cout << y << '\n';
    f();
    std::cout << y << '\n';
    return 0;
}
\end{consolethree}

Using only parameters and return values, you can tell which exactly what information is going in and out of a function. That's a big advantage. The moment you start adding global variables everywhere, things become extremely messy.

\begin{ex}
Study the following program carefully. Rewrite the program so that it \EMPHASIZE{does not} use a global variable (i.e. there's no variable in the global scope.) [Hint: You need to pass the variable using pass-by-reference.]

\begin{consolethree}[escapeinside=||]
#include <iostream>

int count = 0;

void increment()
{
    count++;
}

int main()
{
    for (int i = 0; i < 10; ++i)
    {
        increment();
    }
    std::cout << count << '\n';
    return 0;
}
\end{consolethree}
\end{ex}
